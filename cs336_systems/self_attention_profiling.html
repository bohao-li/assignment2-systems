This section explains the underlying reasons for the performance and memory trends observed in the chart, particularly the critical impact of sequence length.

The most memory-intensive operation in a standard self-attention forward pass is the creation of the attention score matrix, which is computed as a batched matrix multiplication of the queries ($Q$) and keys ($K$). Given that the queries and keys both have the shape of (batch size, sequence length, embedding dimension) or $(B, S, D)$, the resulting score matrix has a shape of $(B, S, S)$. For a backward pass, this score matrix, along with the other intermediate tensors, must be stored to compute the gradients.

The memory for this intermediate score matrix scales quadratically with the sequence length ($O(B \cdot S^2)$). For example, consider the configuration with $B=8, D=128$, and $S=16384$. The score matrix would have a size of $8 \times 16384 \times 16384$. Given that each element is a 4-byte floating-point number (FP32), the memory required for this one tensor alone would be approximately $8 \times 16384 \times 16384 \times 4 \text{ bytes} \approx 8.59 \text{ GB}$. When you factor in the memory for the input tensors, the gradients, and other intermediate values, it's clear why out-of-memory (OOM) errors occur at high sequence lengths, as this quickly exceeds the memory capacity of most GPUs.

The memory saved for the backward pass changes with the sequence length in a non-linear way. As the sequence length increases, the amount of memory needed to store the intermediate activation tensors for the backward pass (especially the $B \times S \times S$ attention score matrix) grows at a much faster rate than the memory for the input tensors. To eliminate this memory cost, techniques such as gradient checkpointing or more advanced methods like FlashAttention are used. Gradient checkpointing re-computes the necessary intermediate tensors during the backward pass instead of storing them, trading computation time for a significant reduction in memory. FlashAttention, on the other hand, reorders the computation of the attention mechanism to avoid storing the large $S \times S$ attention matrix altogether. These methods are crucial for training large language models with long context windows.
